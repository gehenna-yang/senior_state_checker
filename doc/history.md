# 개발 이력 (Project History)

본 문서는 '어르신 상태 체크 앱(senior_check_state)'의 개발 과정과 구현된 기능을 기록합니다.

## 1. 프로젝트 초기 설정 및 아키텍처 구축
- **상태 관리 도입**: `flutter_riverpod` 및 `riverpod_generator`를 사용하여 타입 안전하고 효율적인 상태 관리 기반 마련.
- **클린 아키텍처 적용**: 프로젝트 구조를 `core`, `domain`, `data`, `ui` 계층으로 분리하여 유지보수성 및 확장성 확보.
- **선언적 라우팅**: `go_router`를 도입하고 `StatefulShellRoute`를 통해 탭 간 상태가 유지되는 하단 네비게이션 바 구현.

## 2. 핵심 비즈니스 로직 및 데이터 모델링
- **상태 체크 엔티티**: 투약, 식사, 운동을 구분하는 `CheckItem` 불변 모델 구현 (Freezed 활용).
- **리포지토리 패턴**: 데이터 접근 로직을 추상화하고, 초기 개발 및 테스트를 위한 `MockCheckRepository` 구현.
- **홈 화면 UI**: 어르신 친화적인 큰 글씨와 직관적인 카드 레이아웃 적용.

## 3. 보이스 인식을 통한 간편 상태 체크 (STT)
- **음성 인식 연동**: `speech_to_text` 패키지를 활용한 한국어 음성 인식 서비스 구축.
- **지능형 키워드 매칭**: "약", "식사/밥", "운동/산책" 등의 키워드를 분석하여 자동으로 상태를 업데이트하는 로직 구현.
- **사용자 피드백**: 음성 인식 중 마이크 아이콘 색상 변화 및 처리 결과 안내 스낵바 추가.

## 4. 보호자 데이터 공유 및 관리
- **보호자 모델링**: 보호자 정보와 공유 권한을 관리하는 `Guardian` 엔티티 및 리포지토리 구축.
- **실시간 공유 설정**: 프로필 화면에서 보호자별로 공유 상태(ON/OFF)를 즉시 제어할 수 있는 기능 구현.

## 5. 항목 추가 및 편집 기능
- **관리 화면 구현**: 사용자가 직접 새로운 체크 항목을 등록하거나 기존 항목의 제목, 유형, 예정 시간을 수정할 수 있는 `AddEditItemScreen` 개발.
- **고유 ID 관리**: `uuid` 패키지를 도입하여 각 데이터의 식별성 보장.

## 6. 어르신 맞춤 접근성 강화 (디자인 테마)
- **고대비 테마**: 시력이 약한 어르신들을 위해 배경과 글자의 대비를 극대화한 고대비 모드(Dark Mode 기반) 구현.
- **글자 크기 조절**: 앱 전체 글자 크기를 3단계(보통, 크게, 아주 크게)로 조절할 수 있는 기능 및 설정 영속성(`shared_preferences`) 적용.

## 7. 운동 트레킹 및 건강 리포트
- **운동 기록 고도화**: 운동 시작/종료 시간을 기록하고, 시간을 기반으로 이동 거리(km)를 자동 산출하는 로직 구현.
- **건강 대시보드**: 오늘 하루의 달성률을 시각적 그래프로 보여주는 리포트 화면(검색 탭 개편) 구축.

## 8. 안정성 검증 (Unit Test)
- **단위 테스트 구축**: 핵심 뷰모델 로직, 통계 계산, 음성 명령 처리 등에 대한 테스트 코드를 작성하여 모든 테스트 통과 확인.
